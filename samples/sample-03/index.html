Daily Ops Data Pipeline

Skill: Ops Analyst  ·  Connector: Slack  ·  MCP: Data Pull
Claude.ai subscription required — Requires Claude.ai Pro or Team. Set up the Skill in Claude → Skills, connect the Connector under Settings → Integrations, and install the MCP server via Claude Desktop.
Use Case

Run every morning at 07:00 to pull the previous day's operational metrics, compare them against 7-day baselines, flag anomalies with severity ratings and concrete actions, and deliver a formatted briefing to Slack — so the team starts each day knowing exactly what needs attention.

Skill Definition — Ops Analyst

Copy the skill definition below and paste it into Claude → Skills → New Skill. This gives Claude the rules to analyse metrics, rate anomaly severity, and write plain-English summaries for non-technical readers.

# Skill: Ops Analyst

## Purpose
Analyse a set of daily operational metrics against historical baselines. Flag
anomalies with severity ratings, write plain-English explanations, suggest one
concrete action per anomaly, and produce an executive summary a non-technical
reader can act on in under 60 seconds.

## When to Use This Skill
Use this skill when you have:
- A structured metrics object with today's values
- Baseline values (7-day and/or 30-day averages)
- A list of which metrics matter and what direction is "good"

## Instructions
You are an operations analyst at a fast-growing company. Look at yesterday's
numbers, find what's off, and tell the team what to do about it — in plain
English, in order of priority.

### Analysis Rules
1. Calculate deviation: (today - baseline) / baseline * 100
2. Classify severity:
   - CRITICAL: deviation > ±20%
   - WARNING: deviation > ±10%
   - NORMAL: deviation ≤ ±10%
3. Note direction — UP on support tickets is bad; DOWN on revenue is bad.
4. Look for correlations — orders down 24% AND tickets up 68% are connected.
5. One action per anomaly — specific, actionable within the next hour.
6. Executive summary — 3–5 sentences. Start with most critical. End with
   single priority action.

## Output Format
Return JSON:
{
  "date": "YYYY-MM-DD",
  "overall_status": "NORMAL | WARNING | CRITICAL",
  "anomalies": [
    {
      "metric": "metric_name",
      "value": 0,
      "baseline_7d_avg": 0,
      "deviation_pct": 0.0,
      "severity": "CRITICAL | WARNING",
      "plain_english": "Business-terms explanation.",
      "suggested_action": "Specific action to take in the next 1–2 hours."
    }
  ],
  "normal_metrics": ["metric_a", "metric_b"],
  "executive_summary": "3–5 sentence summary for leadership.",
  "recommended_priority": "Single most important action right now.",
  "analysis_generated_at": "ISO8601 timestamp"
}

Step-by-Step Setup

  1. Create the Skill in Claude
    Go to claude.ai/skills, click New Skill, name it ops-analyst, paste the skill definition above, and save.
  2. Connect the Slack Connector
    Go to Settings → Integrations and connect your Slack workspace. Authorise the chat:write scope so Claude can post the daily briefing to your chosen channel (e.g. #ops-daily).
  3. Install the Data Pull MCP
    Add the data-pull MCP server to your Claude Desktop config (see mcp-config.json). This connects Claude to your metrics sources — PostgreSQL, BigQuery, Stripe, Shopify, or any REST API returning daily stats. Restart Claude Desktop after editing.
  4. Define Your Metrics and Baselines
    Edit the MCP config or data source queries to return today's metrics alongside their 7-day and 30-day averages. Specify which direction is "good" for each metric (higher or lower).
  5. Run the Workflow in Claude
    In Claude, type: "Use the ops-analyst skill. Pull yesterday's metrics via the data-pull MCP for date 2026-06-27. Analyse for anomalies and post the briefing to the #ops-daily Slack channel via the Slack connector."
  6. Schedule Daily Runs
    Use Claude's Projects or a simple cron task to trigger this workflow automatically at 07:00 each weekday. The team will see the briefing before standup every morning.
