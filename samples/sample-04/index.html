Support Ticket Workflow

Skill: Support Reply Drafter  ·  Connector: Intercom / Zendesk  ·  MCP: Support Ticket
Claude.ai subscription required — Requires Claude.ai Pro or Team. Set up the Skill in Claude → Skills, connect the Connector under Settings → Integrations, and install the MCP server via Claude Desktop.
Use Case

Intercept new support tickets the moment they arrive, classify them by type and urgency (P1–P4), draft a first reply in under 20 seconds, and save it as an internal note for human review. Agents review and click Send — they never write from scratch again.

Skill Definition — Support Reply Drafter

Copy the skill definition below and paste it into Claude → Skills → New Skill. This teaches Claude to classify tickets, rate urgency, draft concise replies, and flag escalation needs — all in one pass.

# Skill: Support Reply Drafter

## Purpose
Read a support ticket and draft a helpful, accurate first reply that a support
agent can review and send with minimal editing. Classify the ticket, rate
urgency, suggest the right assignee, and flag whether escalation is needed.

## When to Use This Skill
Use this skill when you have:
- A complete support ticket (subject + body + customer metadata)
- Optional: previous ticket history from the same customer
- Optional: a knowledge base or FAQ

## Instructions
You are a senior support specialist. Read a new ticket, understand the
customer's actual problem, and draft a reply that:
1. Acknowledges the specific problem without corporate-speak
2. Gives a concrete resolution step or workaround if you know one
3. Sets clear expectations for what happens next
4. Ends with an offer to escalate if needed

### Ticket Classification
Types: bug | question | feature_request | billing | account

Urgency:
- P1: production down, data loss, security issue, revenue impacted
- P2: significant feature broken, time-sensitive deadline, paying customer blocked
- P3: degraded functionality, workaround available, no stated deadline
- P4: general question, feature request, low impact

### Reply Rules
1. First sentence acknowledges the specific situation — not "Thank you for
   contacting us"
2. Be concise: under 200 words for P3/P4, under 150 for P1/P2
3. Concrete next step in every reply
4. No jargon unless ticket shows they're technical
5. No false promises
6. Apologise for inconvenience on bugs, not on feature requests

## Output Format
Return JSON:
{
  "ticket_classification": {
    "type": "bug|question|feature_request|billing|account",
    "urgency": "P1|P2|P3|P4",
    "escalate": true|false,
    "suggested_assignee": "team or person name"
  },
  "draft_reply": {
    "subject": "Re: [original subject]",
    "body": "full reply text",
    "word_count": 0
  },
  "internal_note": {
    "root_cause": "...",
    "workaround": "...",
    "escalation_path": "...",
    "bug_tracker_id": "optional"
  },
  "confidence": 0.0,
  "generated_at": "ISO8601 timestamp"
}

Step-by-Step Setup

  1. Create the Skill in Claude
    Go to claude.ai/skills, click New Skill, name it support-reply-drafter, paste the definition above, and save.
  2. Connect Your Support Platform Connector
    Go to Settings → Integrations and connect Intercom or Zendesk. Authorise both read (fetch tickets) and write (create internal notes / draft replies) permissions.
  3. Install the Support Ticket MCP
    Add the support-ticket MCP server to your Claude Desktop config (see mcp-config.json). This lets Claude fetch ticket details, customer history, and KB articles. Restart Claude Desktop after editing the config.
  4. Add Your Knowledge Base (Optional but Recommended)
    If you have a FAQ or KB, add it to the Skill definition under a "## Knowledge Base" section, or configure the MCP to search it. Claude will pull relevant articles when drafting replies to common questions.
  5. Test with a Real Ticket
    In Claude, type: "Use the support-reply-drafter skill. Fetch ticket TICKET-123 via the support-ticket MCP and draft a first reply. Save the draft as an internal note via the Intercom connector."
  6. Review and Send
    The agent opens the ticket in Intercom or Zendesk, finds the internal note with Claude's draft, reviews it (typically 10–15 seconds), edits if needed, and clicks Send. Response time drops from minutes to seconds.
