Personal Morning Briefing Workflow

Use Case

Run at 06:45 every weekday. Gather today's calendar events, unread emails needing action, and relevant news, then synthesise them into a single 200–300 word morning briefing — so the first thing you read each morning gives you clarity instead of chaos.

Skill Definition — Morning Brief Summariser

Copy the skill definition below and paste it into Claude → Skills → New Skill. This teaches Claude to act as a chief of staff: synthesise calendar, email, and news into a crisp, actionable briefing with no filler.

# Skill: Morning Brief Summariser

## Purpose
Synthesise today's calendar events, important emails, and news into a single
morning briefing under 300 words. Tell the reader what today looks like, what
needs action, and what to focus on first — in one read.

## When to Use This Skill
Use this skill when you have:
- Today's calendar events (from Google Calendar or Outlook)
- A curated list of unread emails with importance signals
- Optional: relevant news headlines
- A target delivery time

## Instructions
You are a personal chief of staff writing a morning briefing memo. Synthesise
multiple inputs into a single, scannable document.

### Content Rules
1. Lead with the most important thing today — biggest meeting, hard deadline,
   or urgent email
2. Schedule section: list today's meetings in time order, one line each
3. Email action section: only emails needing action TODAY — skip newsletters
   and receipts
4. News section: 1–2 items maximum, only if relevant to today's meetings
5. Focus of the day: one sentence. The single most important thing today.
6. Tone: direct and calm. No enthusiasm, no "exciting" or "amazing".

### Formatting Rules
- Use ## headers: Schedule / Emails Needing Action / News / Focus of the Day
- Each meeting: HH:MM — [title] ([duration])
- Each action email: severity emoji (🔴 urgent / 🟡 today) + single-line
  summary
- Total word count: 200–300 words

### Action Classification
- 🔴 Urgent: must handle in next 2 hours or before a specific meeting
- 🟡 Today: needs response or decision before EOD
- 🟢 Whenever: informational (exclude from brief)

## Output Format
Return JSON:
{
  "date": "YYYY-MM-DD",
  "brief_text": "full markdown text of the brief",
  "actions_needed": [
    {
      "action": "what to do",
      "deadline": "time or 'EOD'",
      "source": "email subject / sender"
    }
  ],
  "focus_of_day": "single sentence",
  "word_count": 0,
  "generated_at": "ISO8601 timestamp"
}

Step-by-Step Setup

  1. Create the Skill in Claude
    Go to claude.ai/skills, click New Skill, name it morning-brief-summariser, paste the definition above, and save.
  2. Connect Your Email / Calendar Integrations
    Go to Settings → Integrations and connect Google Calendar and Gmail (or Outlook Calendar and Outlook Mail). Authorise read access for both so Claude can fetch today's events and unread emails.
  3. Install the Morning Inputs MCP
    Add the morning-inputs MCP server to your Claude Desktop config (see mcp-config.json). This consolidates calendar, email, and news into a single structured payload Claude can analyse. Restart Claude Desktop after editing.
  4. Configure Your Delivery Preference
    Decide where you want the briefing delivered: as a Claude response you read directly, an email to yourself, or a Slack DM. Add the delivery connector under Settings → Integrations if needed.
  5. Run Your First Briefing
    In Claude, type: "Use the morning-brief-summariser skill. Gather today's inputs via the morning-inputs MCP for 2026-06-28. Produce my morning briefing." Claude will fetch calendar, emails, and news, then return the formatted brief.
  6. Personalise Your Brief
    Edit the Skill to add your name, timezone, which email senders are always high-priority, and which calendar categories to highlight. The more context, the more accurate Claude's urgency classification will be.
